# ProxyGW 深度审核报告（2026-04-14）

审核范围：
- 后端：`/root/proxygw/backend/main.go`
- 前端：`/root/proxygw/frontend/dist/index.html`
- 服务与部署：`systemd/`, `scripts/`, 运行中的 mosdns/xray/proxygw
- 文档：`README.md`, `docs/`

## 一、主要问题（发现）

1) Mosdns 配置生成链路易损
- 历史 patch 导致 YAML 行内拼接错误、插件顺序错乱
- 表现：mosdns 启动失败（mapping/exec missing args）

2) DNS 保存后配置不落盘
- `/api/dns` 仅写数据库，不立即重生成 Mosdns

3) 安全问题
- 登录 token 曾被静态硬编码
- Xray 指定版本更新存在命令注入风险（未校验 version，shell 拼接）

4) 模式切换逻辑不完整
- Mode A/B 切换仅处理 nftables，未同时管理 FRR

5) 工程卫生问题
- backend 目录残留大量临时修复脚本和临时 Go 文件，影响 `go test ./...` 与可维护性

6) 文档与实现不一致
- 根 README 信息过时/夸大
- docs 内容过少，缺 API、运维、故障排查

## 二、本轮已完成修复

1) 恢复 Mosdns 稳定运行
- 修正后端配置生成逻辑
- 验证：`systemctl is-active mosdns` 为 active

2) 修复 `/api/dns` 生效链路
- 保存 DNS 后立即调用 `applyMosdnsConfig()`

3) 恢复版本显示与版本列表
- `/api/status` 返回 `xrayVersion`, `geoVersion`
- 新增/恢复 `/api/xray/versions`

4) 修复更新链路安全性
- `POST /api/update/xray` 增加版本号白名单校验：`^v[0-9A-Za-z._-]+$`
- 移除 shell 拼接下载，改为 `exec.Command("wget", ... )`
- 移除 ghproxy，使用 GitHub 官方地址

5) 修复认证机制
- token 改为进程启动时随机生成（内存态）
- 中间件改为校验 `Bearer <sessionToken>`

6) 修复模式切换
- Mode A: `start nftables`, `stop frr`
- Mode B: `stop nftables`, `start frr`
- 切换后触发 Mosdns/Xray 重生成

7) 清理临时文件
- 将大量临时修复脚本移入 `/root/proxygw/.audit_backup_20260414/backend_tmp`
- 验证：`go test ./...`、`go vet ./...` 可执行通过

## 三、当前状态

- `proxygw`: active
- `mosdns`: active
- 核心页面功能：登录、状态版本显示、版本列表、DNS保存生效 已恢复

## 四、剩余风险与建议（下一阶段）

1) 后端单文件过大
- `main.go` 过重，建议拆分模块：auth/dns/xray/nodes/rules/update/system

2) 自动化测试缺失
- 当前无单元/集成测试，建议优先补：
  - DNS 配置生成快照测试
  - 更新接口参数校验测试
  - 模式切换副作用测试

3) 前端仅 dist 产物
- 建议补充前端源码目录与构建脚本文档，避免线上直接热改 dist

4) 更新链路可观测性
- 建议把组件更新结果、失败原因、耗时写入数据库并前端可视化

5) 权限模型
- 当前是单管理员口令，建议后续引入最小权限与会话过期策略
