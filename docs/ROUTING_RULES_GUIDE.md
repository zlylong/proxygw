# 路由分流规则 (Routing Rules) 新手入门与原理解析

在使用 ProxyGW 时，系统会自动处理复杂的流量分发和 DNS 劫持。但了解背后的“分流规则”（Routing Rules），能帮你更好地理解和排查哪些网站走代理，哪些走直连。

---

## 1. 什么是分流规则？

分流规则是告诉代理网关（内部的 Xray 和 Mosdns）在遇到某个域名或 IP 时，**该走哪条路**的指令簿。

通常有三个核心动作（Outbounds）：
1. **🚀 代理 (Proxy)**：流量被加密，发往海外的节点（用于访问国际互联网）。
2. **🎯 直连 (Direct)**：流量不经过节点，直接从本地网络发出（用于访问国内网站及内网设备）。
3. **⛔ 拦截 (Block)**：流量被直接丢弃（用于屏蔽广告、恶意追踪网站）。

---

## 2. GeoSite 与 GeoIP (底层数据源)

手写成千上万个域名是不现实的，所以我们使用了社区维护的 **GeoData (地理数据包)**：
- **`geosite.dat`**：包含了海量域名的分类集合（如 `geosite:google`, `geosite:cn`）。
- **`geoip.dat`**：包含了 IP 段的分类集合（如 `geoip:telegram`, `geoip:cn`）。

ProxyGW 会定时从 GitHub 自动同步最新的 `v2ray-rules-dat`，你无需手动更新，即可保持分流名单的最新状态。

---

## 3. 常见规则标签解释

在 ProxyGW 面板或配置中，你经常会看到以下标签：

### 🇨🇳 国内直连白名单
- `geosite:cn`：包含了绝大多数中国大陆公司的域名（淘宝、微信、B站等）。
- `geoip:cn`：分配给中国大陆的所有 IP 地址段。
- **处理方式**：必须强制走 **Direct (直连)**。否则访问国内服务会绕一圈海外，导致延迟极高甚至被风控封号（如微信防盗号机制）。

### 🌍 海外代理名单
- `geosite:geolocation-!cn`：不在中国大陆的绝大多数知名海外网站。
- `geosite:google`, `geosite:telegram` 等：特定公司的集合。
- **处理方式**：走 **Proxy (代理)**。

### 🛑 广告屏蔽名单
- `geosite:category-ads-all`：各类广告联盟、数据收集器的域名。
- **处理方式**：走 **Block (拦截)**，在 Mosdns 层面直接返回 0.0.0.0，减少网络请求，加快网页加载。

## 4. 三种工作模式下的分流差异

ProxyGW 支持三种极其强大的透明路由模式，它们的分流逻辑在底层有所不同：

### 🔴 Mode A (全局 TProxy 透明网关)
**“宁可错杀，绝不放过”**
所有的设备流量（不管三七二十一）全被强行塞进 Xray 的 TProxy 入口。Xray 内部会通过 `geosite:cn` 和 `geoip:cn` 规则去判断。
- **优点**：不需要主路由器支持任何高级功能（如 OSPF）。
- **缺点**：国内流量也会去网关内核“旅游”一圈，虽然后面直连了，但会白白消耗 ProxyGW 的软硬件性能。

### 🟢 Mode B (混合 Fake-IP OSPF 旁路)
**“最完美的性能平衡”**
当设备发起 DNS 请求时，内部的 Mosdns 会查规则：
- 针对**域名**，Mosdns 会返回一个 **伪造的 IP (`198.18.x.x`)**。由于 OSPF 提前宣告了这个网段，流量被精准吸入 ProxyGW 进行代理。
- 针对你添加的**具体的规则 IP 或 GeoIP**，系统底层同时会将真实的网段宣告给 OSPF。
- **优点**：性能极高，国内流量完全不经过 ProxyGW，而且同时兼容了域名伪造拦截与指定真实 IP 拦截，补齐了各种网络场景的短板。

### 🔵 Mode C (真实 IP OSPF 动态路由与追踪)
**“用真实 IP 智能引流并动态追踪”**
依然依赖 Mosdns 解析真实 IP，不破坏任何域名的真实性。后端会将 GeoIP 等真实的网段瞬间通过 OSPF 推送到主路由。
- **独家自愈追踪**：如果你在这个模式下添加了某个域名作为规则，后端守护进程会全自动定期解析它的真实 IP，如果该网站的 CDN 或者服务器 IP 换了，系统会无缝平滑地将新 IP 宣告给主路由 OSPF，旧 IP 自动移除。
- **优点**：既保持了 IP 的真实性（避免了 Fake-IP 引起的安全校验拦截），又解决了静态路由无法对付云时代 CDN 变动的问题。

---

## 5. 自定义规则建议

如果你发现某个网站打不开，或者国内网站变慢了，可能是规则误判。你可以通过面板添加**自定义直连域名**。

**最佳实践：**
1. **宁缺毋滥**：只把确定需要代理的走代理，不要随便开启全局代理。
2. **内网直连**：`192.168.0.0/16`, `10.0.0.0/8` 等局域网 IP 永远应该走 Direct 直连，否则可能导致无法访问打印机、NAS 等局域网设备。
